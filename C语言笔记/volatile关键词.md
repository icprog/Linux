

```c
1、
int i=10;
int j=i;
int k=i;
编译器会对代码进行优化。
由于在2个赋值语句之间 i 没有被当做过左值，这时编译器认为 i 的值没有发生变化。
在第1条语句编译器从内存中取出 i 的值给 j 赋值之后，这个值没有被丢弃，而是在第2条语句继续用这个值赋值给 k 。
编译器不会生成汇编代码重新从内存中取 i 的值，这样能够提高效率。
但要注意 i 在2条语句之间没有被当做过左值才行。

2、
volatile int i=10;
int j=i;
int k=i;
volatile告诉编译器 i 的值随时都可能变化，编译器每次使用 i 都必须从内存中取出它的值才行。

3、什么时候使用volatile？
修饰一个寄存器变量。
修饰一个端口的数据。
修饰多个线程共享的数据。
```
